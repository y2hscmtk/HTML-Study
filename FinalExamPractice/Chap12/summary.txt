초기 웹 저장소

웹 서버에 저장=>웹 서비스 중간에 발생하는 일시적 데이터를 저장
문제점 : 웹 사용의 폭발적 증가, 웹 서버 저장용량에 대한 부담
웹 브라우저와 웹 서버 사이의 통신 트래픽 증가

HTML5의 웹 저장소
사용자의 로컬 컴퓨터에 일부 데이터 저장
=> 웹 서버의 저장 용량 및 통신 트래픽 감소
저장소 종류
: 쿠키,웹 스토리지,로컬 파일,인덱스트 데이터베이스
=> 웹 서버와 연결이 끊어진 경우에도 로컬 컴퓨터에도 웹 애플리케이션 실행

쿠키
웹 서버가 브라우저에게 지시하여 로컬 컴퓨터에 저장하는 작은 데이터
쿠키는 웹 서버가 생성하여 브라우저로 보맨
사용자가 어떤 웹 서버에 처음 접속할때
웹 서버가 다음 요청에서 그 사용자들을 기억할수 있또록 쿠키를 만들어 전송
쿠키를 받은 브라우저는 컴퓨터에 저장
동일한 웹 서버에 요청할시 쿠키를 함께 전송
쿠키를 받은 웹 서버는 어떤 사용자로부터 요청이 왔는지 파악 가능(사용자 기억)
브라우저가 웹 서버로 쿠키를 전송할때는 값만 전송한다

웹 스토리지
: 웹은 웹 애플리케이션의 형태로 진화
실행 도중 로컬 컴퓨터에 데이터 저장 공간 필요 => 양이 많기 때문에 쿠키에는 저장할수 없음
=>쿠키는 불 필요한 트래픽 발생
==> 쿠키는 윈도우마다 독립적인 값을 저장 불가
=> 웹 스토리지 도입 : 로컬 컴퓨터으 ㅣ저장공간, 웹 서버의 저장 부담과 네트워크 트래픽 감소
=> 자바 스크립트로만 읽고 쓸수있다.

웹 스토리지 : 세션 스토리지& 로컬 스토리지 2가지의 종류로 나뉜다
특징 : 문자열만 저장, (키,값)으로 구성도니 아이템 단위로 저장
동일한 키를 가진 아이템은 존재할수 없음
키와 값은 대소문자 구분
저장 검색 삭제 등 웹 스토리지의 조작은 모두 자바스크립트 코드로 작성된다.

세션 스토리지의 생성과 소멸
세션 : 연결된 웹 사이트와 윈도우 (웹 페이지와 연결됐을때 생성됨)
윈도우에 웹 사이트가 로드 될때 세션 스토리지 생성
한 윈도우에 여러 세션 스토리지 생성 가능 => 여러 웹 사이트를 열수 있기때문
=> 다른 윈도우 창에따라 다른 세션 스토리지가 생성됨
윈도우가 닫히면 세션 스토리지가 없어짐
